# keel_api_key (Resource)

Manages a Keel API key. API keys are immutable after creation — any change forces replacement. Deletion revokes the key.

Use this resource to create API keys for the project associated with the provider API key. The provider API key must have permission to create and revoke keys. API keys are immutable after creation; changing configurable fields forces replacement, and deletion revokes the key.

## Example Usage

```terraform
resource "keel_api_key" "backend" {
  name        = "backend-service"
  description = "Key for the backend service"
  scope       = "client"
}

output "api_key_id" {
  value = keel_api_key.backend.id
}

output "api_key_project_id" {
  value = keel_api_key.backend.project_id
}

output "api_key_prefix" {
  value = keel_api_key.backend.prefix
}
```

## Import

Import with `key_id` or `project_id/key_id`.

```shell
terraform import keel_api_key.backend key_id
terraform import keel_api_key.backend project_id/key_id
```

## Schema

### Optional

- `description` (String) API key description.
- `expires_at` (String) Expiration timestamp, if configured.
- `name` (String) API key name.
- `project_id` (String) Project ID this key belongs to. When omitted, the provider uses /v1/api-keys and the Keel API derives the project from the provider API key.
- `scope` (String) Key scope: "admin", "client", or "approval". Default: "admin".

### Read-Only

- `created_at` (String) Creation timestamp.
- `created_by` (String) User ID that created the API key, if available.
- `id` (String) API key ID.
- `last_used_at` (String) Last-used timestamp, if the key has been used.
- `prefix` (String) Key prefix (visible portion).
- `raw_key` (String, Sensitive) The raw API key. Only available at creation time.
- `revoked_at` (String) Revocation timestamp, if the key has been revoked.
- `status` (String) Derived key status: "active" or "revoked".
